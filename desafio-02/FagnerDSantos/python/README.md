#  1 - DESAFIO 02 – NÚMEROS PRIMOS

    1.1 - SEGUI UM RACIOCÍNIO UTILIZANDO A REGRA MATEMÁTICA DE FORMA PURA. SABEMOS QUE 0 E 1 NÃO PODEM SER NÚMEROS PRIMOS, ENTÃO DEFINI UM LOOP QUE PERCORRE DE 2 A 10000, IDENTIFICANDO SE O NÚMERO SÓ PODE SER DIVIDIDO POR 1 OU POR ELE MESMO.

    1.2 - SEGUINDO NESSA LINHA DE RACIOCÍNIO, ASSUMI INICIALMENTE QUE TODOS OS NÚMEROS ERAM PRIMOS. MAIS ABAIXO, FIZ UMA REGRA DE DIVISÃO UTILIZANDO O OPERADOR % (MÓDULO), NO QUAL, SE FOR RETORNADO 0, SIGNIFICA QUE O NÚMERO POSSUI UM DIVISOR E, PORTANTO, NÃO É PRIMO.

    1.3 - FOI ADICIONADO UM BREAK PARA INTERROMPER O LOOP QUANDO UM DIVISOR FOR ENCONTRADO, OU SEJA, SE FOI ENCONTRADO UM DIVISOR, NÃO HÁ MOTIVO PARA CONTINUAR TESTANDO, POIS JÁ SABEMOS QUE O NÚMERO NÃO É PRIMO.

    1.4 - APÓS ESSAS DEFINIÇÕES, O CÓDIGO PERCORRE UM RANGE DE 2 A 10000 E IDENTIFICA QUAIS NÚMEROS SÃO PRIMOS.

# 2 - UTILIZAÇÃO DO PROGRAMA

    2.1 - O PROGRAMA PODE SER UTILIZADO POR TERCEIROS PARA IDENTIFICAR NÚMEROS PRIMOS DENTRO DE UM INTERVALO DEFINIDO NO CÓDIGO.

    2.2 - PARA UTILIZÁ-LO, BASTA TER O PYTHON INSTALADO NO COMPUTADOR E EXECUTAR O SCRIPT EM UM TERMINAL OU EM UMA IDE, COMO VS CODE, PYCHARM OU JUPYTER NOTEBOOK.

    2.3 - O USUÁRIO TAMBÉM PODE ALTERAR O INTERVALO DE NÚMEROS NO RANGE, CASO DESEJE BUSCAR NÚMEROS PRIMOS EM UM INTERVALO MAIOR OU MENOR.
